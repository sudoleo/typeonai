Configuration

Per-provider model lists shown in the app's model pickers (row order = picker order). Default sets the Free-tier preselection. Premium marks Pro-only models. Consensus offers the model in the Consensus picker. Models marked Required are enforced server-side and re-added automatically — that is why some entries can't be removed here.

Preset Model Sets

Each preset selects all six answer models and the final Consensus engine together. Fast and Balanced accept Free-tier models only; High Quality is always Pro-only.

Deep Think Model

Consensus engine Deep Think switches to while active. Deep Think remains a separate request mode: it adds the deeper prompt, reasoning settings, token budget and quota on top of the preset selection. The selected engine stays in the Consensus list.

Differences Judges

Judge models per provider family for the Differences engine and the Resolve round. Standard engines get the Standard judge, Pro engines the Pro judge (always run at low effort). Frontier-Low IDs are internal aliases and not selectable.

Judge assignment

Which family judges which Consensus engine. The judge family is always a different one than the engine's (anti self-judging). Auto picks the first family with an available API key from the priority order; a manual pick is preferred first and falls back to Auto if its key is unavailable.

Consensus Engines

Ordered list shown in the app's Consensus picker. Aliases (e.g. Gemini-Pro) resolve to a provider model server-side and follow config updates automatically; direct model IDs must exist in a provider list on the Models tab. The first entry is preselected for new users.

Request, input, and token limits for Free and Pro users. Limits apply after the daily reset.

Scheduled Consensus Publisher

Controls the GitHub Actions publishing run from Firestore. Generated titles are checked for a short, single Google search intent before a Consensus run starts. New pages are generated Monday, Wednesday, and Friday by the scheduled workflow. Every generated Watch is weekly and permanently pinned to the configured Free Watch providers, regardless of the owner account tier. DeepSeek is excluded from both the initial Publisher run and every Watch rerun.

weekly · free Watch providers · DeepSeek excluded

Consensus API keys

Issue a key for an active, e-mail-verified Firebase user. The key inherits that UID's tier and run limits. Normal v1 runs use all six providers, including mandatory DeepSeek processing in China; Scheduled Publisher runs explicitly exclude DeepSeek. Run content expires after 30 days. Only a SHA-256 hash is stored; the full key is shown exactly once and should be copied directly into a secret manager.

Automation Watch pages

Pages created by the Scheduled Consensus Publisher with its permanently pinned weekly Free-model Watch. Deleting a page also removes its Watch schedule, Watch history, and follower subscriptions.

Shared Pages Moderation

Reported pages first (review-priority, then report count). Index puts a page into sitemap-shares.xml and removes its noindex — only do this for eligible pages. Delete permanently removes the page and all Watch-bound data.

Consensus Watch diagnostics

Test SMTP independently or start an active watch immediately through the normal leased scheduler path. Manual runs make real model calls, write history, advance their schedule, and apply their configured e-mail rule.
